llmindex

A machine-readable index standard for LLM and AI search discovery. Tell AI agents what your website offers and where to find it.

GitHub Repository Read the Spec

What is llmindex?

AI search engines (ChatGPT, Perplexity, Gemini) answer user questions by reading website content directly. But there's no standard way for websites to tell AI agents what information is available and where to find it.

llmindex defines a lightweight JSON manifest at /.well-known/llmindex.json — think of it as robots.txt for the AI era.

How it works

LLM visits your site
→
Fetches /.well-known/llmindex.json
→
Reads /llm/* endpoints
→
Answers accurately

Structured Discovery

JSON manifest with typed schema. AI agents know exactly where your products, policies, and FAQ live.

Machine-Readable Feeds

JSONL product feeds let AI ingest your catalog as structured data, not scraped HTML.

Domain Verification

Built-in DNS TXT or HTTP file verification proves you own the domain.

Version Control

Timestamped, versioned manifests. AI agents can detect stale data and re-crawl.

Quick Start

Create /.well-known/llmindex.json on your server:

{
  "version": "0.1",
  "updated_at": "2026-02-20T00:00:00Z",
  "entity": {
    "name": "Your Brand",
    "canonical_url": "https://your-site.com"
  },
  "language": "en",
  "topics": ["your-industry"],
  "endpoints": {
    "products": "https://your-site.com/llm/products",
    "policies": "https://your-site.com/llm/policies",
    "faq": "https://your-site.com/llm/faq",
    "about": "https://your-site.com/llm/about"
  }
}

Then create the /llm/* pages as Markdown or HTML. That's it.

Install the tools

Python CLI (Generator + Validator)

pip install llmindex

# Generate from CSV
llmindex generate \
  --site "My Store" \
  --url https://mystore.com \
  --input-csv products.csv

# Validate
llmindex validate dist/.well-known/llmindex.json

npm (Schema + TypeScript Types)

npm install @llmindex/schema

// Validate with Ajv
import Ajv from "ajv";
import schema from "@llmindex/schema";

const ajv = new Ajv();
const validate = ajv.compile(schema);
validate(manifest); // true/false

Specification

Required Fields

FieldTypeDescription
versionstring"0.1"
updated_atstringISO 8601 datetime
entity.namestringBrand/company name
entity.canonical_urlstringHomepage (HTTPS)
languagestringBCP-47 code
topicsarrayCategory tags (1+)
endpointsobjectproducts, policies, faq, about

Optional Fields

FieldDescription
feedsMachine-readable data feeds (JSONL, JSON)
verifyDomain ownership proof (DNS/HTTP)
sigCryptographic signature (JWS)
licenseSPDX identifier or URL
